Mecanica de execução 

O codigo delph é estruturado então existem sessões que devem ser executadas em uma ordem 
para que funcionem 

1 - program Projeto -> isso é arquivo do porjeto que da a referencia para ele começar 

2 - uses -> é onde são declaradas ass classes e bibliotecas que vão ser usadas para a execcução 

3 - begin -> aqui fica o codigo que vai ser executado é onde a aplicação recebe o comando de inicio
   
4 - end   -> aqui ele sabe que execução termina então a logica deve estar dentro de begin e end
