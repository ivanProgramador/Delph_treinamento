para se conectarrr a base de dados o Delph precisa de uma DLL,
chamada libmysql.dll essa dll deve ficar dentro da pasta windows
oque faz a conexão com o banco d dados e uma componete chamado 
data module.

para criar um data module
new / other / database / Data Module    

embora pareça ele não é um formulário comum ele não fica visivel
na hora de executar e ao onves de receber componetes como input , label ...,
ele recebe componetes que se conetam a base de dados 

componentes de conexão 
esses componentes ficam na paleta FireDac (Data Access Components)

1 - componente FDConnection (admin conexão)
2 - componente FDTable (admin tabela)
3 - componente DataSouce da paleta DataAccess (admin campos)

configgurando a conexão
1 - clica no  FDConnection vai aparecer um formulario 
2 - seleciona o driver mysql 
3 - no database coloca o nome do banco, preenche usuario e  senha.

depois clica na propridade conected ele vai pedir a senha eo login do banco
mas isso não pode ficar dessa forma porque senão toda avez que algo for reuisitado da base de dados ele trava o
o sistema e pede de novo, pra concertar isso tem que ir na prop LoginPronpt e colocar false

com a configuração de conexão pronta agora eu preciso conectar isso a interface grafica que eu criei 

para fazer isso eu tenho que usar so componentes presentes na paleta data control.

"DBEdit1" é um componente de entrada porém ele se coneta a base de dados difrente de um campo de entrada comum
o componente DM não está ligado a tela então no menu inicial onde tem a aopção new tem uma opção chamado useUnit 
onde podemos fazer a abusca e fazer um import dessa unidade para dentro da interface grafica, assim eu consigo conectar 
o "DBEdit1" na base de dados existem 2 configurrações de orintação pra ele.
 
1 - data source vai apontar para tabela 
2 - data field vai apontar para o campo da tabela ao qual ele se refere 

com os campos configurados então eu coloquie um componete chamado TDBnavigator que possui alguns botões que possa usar para 
navegar dentro da base de dados.ele tambem tem o data source para se conectar a tabela.

Tabela que lista os dados é um componente chamado TDBGrid ela puxa e organiza so dados na tabela de forma automática.
caso tenha alguma coluna da tabela qquan não precise aparecer bas ablicar no componet duas vezes vai aparecer uma menu
clique no botão add all fields para puxar todos as colunas e delete a coluna que você não deseja mostrar
